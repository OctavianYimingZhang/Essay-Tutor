# Drafting And Critical Analysis

Use this reference after plan approval or when revising essay prose. This file controls paragraph logic, critical analysis, language discipline, major revision, and capacity reallocation.

## Draft From The Approved Plan

The approved plan controls:

- thesis;
- scope;
- section hierarchy;
- paragraph functions;
- citation strategy;
- figure/table/data strategy;
- critical-analysis targets;
- word-count allocation.

If research after approval contradicts the plan, pause drafting and present a revision note rather than silently changing the essay.

## Paragraph Logic

Default paragraph:

```text
Claim -> Mechanism/Evidence -> Interpretation -> Scope or Limitation -> Link back
```

Evidence-heavy paragraph:

```text
Evidence -> Mechanism tested -> Result -> Interpretation -> Limitation
```

Comparison paragraph:

```text
Shared problem -> Comparison axis -> Model A -> Model B -> Evidence -> Evaluation
```

Application paragraph:

```text
Problem -> Mechanism or theory -> Intervention or case -> Evidence -> Boundary -> Implication
```

Each paragraph should have one main function. Do not use a paragraph as a container for loosely related facts.

## Critical Analysis Framework

Every substantive body paragraph must perform at least one critical-analysis task:

- evidence strength: state what the evidence can and cannot support;
- alternative explanation: identify another plausible interpretation and why it is weaker, stronger, or unresolved;
- scope or limitation: define the conditions, population, method, period, model, or concept boundary;
- causal calibration: use causal wording only when the cited design supports causality;
- mechanism comparison: compare competing mechanisms or explain why one mechanism is more relevant to the question;
- method limitation: explain how measurement, design, sample, model, or source type affects the claim;
- implication: state the theoretical, clinical, social, or methodological consequence that follows from the evidence.

A paragraph fails if it only describes a mechanism, paper, concept, method, or example. It must explain what the material proves, what it limits, and how it answers the essay question.

## Descriptive And Analytic Balance

Minimum target:

```yaml
AnalyticMinimum:
  introduction: 20_percent
  main_body: at_least_30_percent
  discussion: at_least_70_percent
  conclusion: 50_percent
```

Preferred whole-essay balance:

```yaml
PreferredEssayBalance:
  descriptive: 40_to_60_percent
  analytic: 40_to_60_percent
```

Descriptive content:

- defines terms;
- states mechanisms;
- reports evidence;
- describes methods;
- gives source-backed examples.

Analytic content:

- explains why a mechanism matters;
- states what evidence proves or fails to prove;
- compares models;
- defines scope or limitations;
- calibrates causal strength;
- links evidence to the essay question;
- identifies clinical, theoretical, social, or methodological implications.

A paragraph fails if it has more than two consecutive descriptive sentences without analysis.

## Section Rules

Introduction:

- start with the problem or thesis, not metacommentary;
- define only necessary terms;
- set the boundary of the essay;
- state the argument directly;
- avoid listing every body section when a direct thesis and organising logic are enough.

Main body:

- use subtitles when they improve navigation;
- make each subtitle an argument unit, not a topic label only;
- integrate evidence and interpretation next to each other;
- prevent mechanism catalogues by explaining what each mechanism proves or distinguishes;
- when models conflict, rank them by explanatory strength only when the evidence supports that ranking.

Discussion:

- synthesise the main body rather than repeating it;
- compare the explanatory strength of the essay's main claims;
- evaluate evidence quality, model fit, method limits, uncertainty, and alternative explanations;
- define what can and cannot be claimed from the evidence base;
- identify future directions only when they follow from a named uncertainty or limitation.

Conclusion:

- answer the question directly;
- synthesize the thesis and implications;
- add no new evidence, source, mechanism, or claim.

## Language Rules

Use academic prose that is direct, precise, and evidence-calibrated.

Required:

- Start with the answer, problem, or thesis.
- Prefer direct positive claims.
- Keep model, mechanism, evidence, limitation, and implication distinct.
- Use examples, experiments, figures, and cases as evidence slots, not standalone mini-summaries.
- Convert studies into evidence, interpretation, and limitation.
- State aims as analytical tasks when writing lab reports: measured signal, calculated parameter, comparison, and intended interpretation.
- Compress repeated definitions, repeated claims, decorative transitions, and low-value background.
- Reallocate deleted capacity to mechanism, interpretation, limitation, alternative explanation, evidence strength, scope, or direct link-back.
- Replace first-person process narration and software-button narration with reproducible method logic and data-analysis rationale.
- Preserve causal strength, scope qualifiers, and model boundaries when shortening.
- Cite non-obvious factual claims, theories, mechanisms, experiments, data, or generalisations.

Avoid:

- lecture-route, source-route, or author-route narration;
- process commentary inside the essay body;
- decorative transitions;
- broad claims such as "this is important" without a specified consequence;
- unsupported statistics;
- mechanism inferred from a graph, fitted line, or trend unless the design tests that mechanism;
- citation stacking;
- author-led citation wording when parenthetical citation would support a content-first claim more cleanly;
- negative-only openings that are immediately corrected in the next sentence;
- detail catalogues that do not explain what the listed items prove or distinguish;
- calculation working that replaces interpretation;
- inconsistent notation, units, variable names, or parameter names after first definition;
- figure or table prose that repeats information better carried by the caption or table;
- true but unneeded detail that makes the answer encyclopedic rather than argumentative.

## Revision Discipline

Examples, teacher edits, feedback, and prior essays may teach only transferable writing behaviour:

- structure;
- density;
- paragraph function;
- revision priorities;
- wording discipline;
- QA checks.

They must not supply factual claims, citations, dates, named mechanisms, lecturer preferences, or topic-specific conclusions for a new essay.

Delete or merge content when it is true but does not improve the answer's function.

Common deletion targets:

- title or heading language that tries to contain the whole essay argument;
- historical setup, discovery narrative, or source-route narration that does not change the thesis;
- repeated definitions, repeated aims, repeated method descriptions, and repeated result statements;
- objectives or section previews that duplicate the introduction or methods;
- first-person process narration about what the writer did with data;
- instrument-button instructions when reproducible scientific configuration is what matters;
- raw calculation working when it replaces interpretation;
- broad intensifiers when the consequence is not specified;
- author-led literature narration when a content claim with parenthetical citation is cleaner;
- long lists of regions, molecules, components, methods, variables, or statistics that do not distinguish a mechanism or interpretation;
- figure or table prose that repeats what the caption or table can carry more efficiently;
- limitation-only discussion paragraphs that do not test alternative explanations, evidence strength, or claim scope;
- unsupported causal wording where the evidence supports only association, consistency, implication, or partial support.

Large deletion is justified only when the removed material consumes capacity without adding one of these functions:

```text
claim
mechanism
evidence
interpretation
scope
limitation
alternative explanation
methodological consequence
link back to the question
```

## Sentence-Level Revision Rules

Prefer these transformations:

| Weak pattern | Revision target |
| --- | --- |
| Metacommentary opener | Direct claim, problem, or thesis |
| "This is important" | Named consequence or evidential function |
| Author-led source narration | Content claim plus parenthetical citation, unless literature history is the point |
| Broad causal verb | Evidence-calibrated verb such as supports, suggests, is consistent with, or constrains |
| Vague evidential verb | Specific scientific verb matched to evidence strength |
| Negative opener followed by correction | Direct positive claim with a compact boundary if needed |
| Setup sentence, repeated claim, then result | Setup plus result, then interpretation |
| Dense list | Higher-level mechanism, unless listed items distinguish the argument |
| Result description only | Result plus what it proves, fails to prove, or leaves ambiguous |
| First-person calculation narrative | Impersonal analysis statement plus formula, value, and interpretation |
| Trendline claim without fit quality | Model, fit metric when available, and calibrated interpretation |

Avoid AI-sounding polish that inflates rather than specifies. Replace general prestige words with concrete function: mechanism, evidence, limitation, consequence, comparison, or scope.

Use hedging only when evidence requires it. Empty caution weakens the argument; overconfident wording creates false certainty. Once a technical term is defined, keep terminology consistent instead of re-explaining it with near-synonyms.

## Capacity Reallocation Pass

Before finalizing a revision, run this pass:

```yaml
CapacityReallocationPass:
  protected:
    - thesis
    - source-backed mechanism
    - essential evidence
    - required method or result detail
    - measurement principle, control, calibration, or model-fit detail
    - interpretation
    - limitation or alternative explanation
    - scope qualifier
  delete_first:
    - repeated framing
    - decorative transitions
    - source-route narration
    - unsupported intensifiers
    - duplicated aims or objectives
    - first-person process narration
    - software-button narration
    - arithmetic steps without interpretation
    - unneeded historical setup
    - detail catalogues without inference
    - table or figure repetition in prose
  add_or_expand_only_if:
    - it sharpens the thesis
    - it links evidence to interpretation
    - it calibrates claim strength
    - it supplies a missing alternative explanation
    - it defines the scope of the conclusion
    - it reports model fit or regression quality needed for rigor
    - it connects a quantitative result to the research aim
    - it updates an outdated evidence base
```

If a requested shortening would remove protected material, state the largest safe reduction and preserve the argument.

## Critical Thinking And Revision QA

Before finalizing, verify that the essay passes:

```yaml
CriticalThinkingAndRevisionQA:
  thesis_is_arguable: true
  evidence_is_interpreted: true
  every_body_paragraph_has_analysis_task: true
  mechanisms_are_not_only_described: true
  models_are_compared_where_relevant: true
  limitations_affect_the_argument: true
  causality_is_calibrated: true
  alternative_explanations_considered: true
  implications_follow_from_evidence: true
  discussion_synthesises_and_evaluates: true
  no_metacommentary_or_source_route_narration: true
  deleted_capacity_reallocated_to_argument: true
  no_true_but_unneeded_detail_catalogues: true
  data_model_and_fit_quality_reported_when_relevant: true
  method_written_for_reproducibility_not_button_clicks: true
  calculations_support_interpretation_not_replace_it: true
  scope_of_conclusions_explicit: true
  conclusion_has_no_new_evidence: true
```

If critical thinking is weak, do not add a detached critical sentence at the end. Rewrite the paragraph so evidence, interpretation, limitation, and link-back are adjacent.
