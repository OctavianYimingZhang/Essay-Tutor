# Essay Tutor Website Coursework

Use this focused skill for academic websites and interactive coursework outputs.

## Shared Resource Rule

Read shared resources from the first existing Essay Tutor root:

- packaged path: `../../`
- local copied-skill path: `../essay-tutor/`
- default local path: `~/.codex/skills/essay-tutor/`

## Workflow

1. Read `references/intake-and-planning.md` and `references/visuals-tables-data.md`.
2. If website output mode, interaction model, or platform constraints are unresolved, use `python3 scripts/build_intake_questions.py website-plan`.
3. Build an academic content architecture before implementation: audience, user journey, sections or routes, interactions, evidence assets, citations, accessibility, and submission constraints.
4. Use interaction only when it helps the academic argument, comparison, explanation, data exploration, or assessment purpose.
5. Read `references/drafting-and-critical-analysis.md` before drafting website copy.
6. Use a web-app or frontend skill only when the user asks to build implementation files.
7. Read `references/qa-and-validation.md` before final delivery.

Do not treat a website assignment as a generic landing page.
